《大模型应用开发》（LLM Application Development）

行业通识（基于Andrew Ng课程/OpenAI文档/LangChain实践）

“AI不再是模型研发的专利，而是应用开发的基石。“

背景介绍

领域概况 随着ChatGPT的发布，AI开发的范式发生了根本性的转变。从传统的”训练模型”（Training Models）转向了”应用开发”（Building Applications）。开发者不再需要精通复杂的数学和算法，只需要掌握提示工程（Prompt Engineering）和编排工具（如LangChain），就能构建出强大的AI应用。

核心资源 本笔记综合了吴恩达（Andrew Ng）的《ChatGPT Prompt Engineering for Developers》系列课程、OpenAI官方文档以及LangChain等开源社区的最佳实践。它是当前大模型应用开发的”实战指南”。

影响力 这一新的开发范式降低了AI的门槛，引发了”AI原生应用”（AI Native Apps）的爆发。无论是构建聊天机器人、智能助手，还是文档分析工具，大模型应用开发都已经成为程序员的必备技能。

核心范式

1. 提示工程（Prompt Engineering）

  • 定义： 用自然语言与模型对话，引导模型输出想要的结果。
  • 原则： 清晰明确、给模型思考时间（CoT）。

2. 检索增强生成（RAG）

  • 解决痛点： 幻觉（一本正经胡说八道）和知识过时。
  • 原理： 外挂知识库。先去知识库里搜相关信息，再把信息喂给大模型，让它”基于这些信息”回答问题。

3. 智能体（Agents）

  • 定义： 让大模型拥有”手”和”脚”。
  • 能力： 规划（Planning）、记忆（Memory）、工具使用（Tool Use）。

主要内容

1. 提示工程的两大原则

  • 编写清晰、具体的指令：
    • 使用分隔符： 用```, """, < >等符号把输入内容隔开，防止提示注入。
    • 结构化输出： 要求模型输出JSON或HTML格式，方便程序解析。
    • Few-Shot（少样本学习）： 给模型几个例子（“教它做事”），它学得更快。
  • 给模型思考的时间（思维链 CoT）：
    • 不要直接问结果，要问”请一步步思考”。
    • 例子： 问数学题，如果直接问答案，它可能算错；如果让它”先列出公式，再代入数值，最后计算”，准确率会大幅提升。

2. RAG（Retrieval-Augmented Generation）架构

这是目前企业级应用最主流的架构：

  1. 文档切片（Chunking）： 把长文档切成小块。
  2. 向量化（Embedding）： 把文本块变成向量（一串数字），存入向量数据库（Vector DB）。
  3. 检索（Retrieval）： 用户提问时，把问题也变成向量，在数据库里找最相似的文本块。
  4. 生成（Generation）： 把找到的文本块作为”上下文”（Context），连同问题一起发给大模型。

3. LangChain与编排

  • 痛点： 直接调API很麻烦，要处理上下文、记忆、格式转换。
  • LangChain： 一个胶水层框架。
    • Chains： 把多个步骤串起来（比如：先总结，再翻译，再发邮件）。
    • Memory： 让模型记住之前的对话（短期记忆/长期记忆）。
    • Tools： 给模型配备计算器、搜索工具、数据库接口。

4. 智能体（Agents）：通往AGI的雏形

  • ReAct模式： Reason（推理）+ Act（行动）。
  • 流程：
    1. 用户： “帮我订一张明天去北京的机票。”
    2. 模型思考： “我需要先查明天去北京的航班。”
    3. 模型行动： 调用”航班查询API”。
    4. 模型观察： 拿到航班列表。
    5. 模型思考： “用户没说具体时间，我得问他。”
    6. 模型回答： “明天有这几班…您想订哪班？“

核心观点

  1. 模型是推理引擎，不是知识库： 不要指望模型记住所有知识（它会忘，也会瞎编）。要把知识放在外部（RAG），只让模型负责处理知识。
  2. 垃圾进，垃圾出（Garbage In, Garbage Out）： 提示词写得烂，模型回答就烂。Prompt Engineering是新时代的编程语言。
  3. 迭代开发： AI应用不是一次写好的，而是”聊”出来的、测出来的。建立评估体系（Eval）非常重要。

应用指导

1. 个人提效

  • 场景： 写周报、读论文。
  • 应用：
    • 总结助手： 把长文扔给AI，让它”用3个bullet points总结核心观点”。
    • 润色助手： “请用更专业的职场语气重写这段话。“

2. 企业知识库

  • 场景： 客服机器人。
  • 应用：
    • 搭建RAG系统。把公司的产品手册、FAQ向量化。
    • 客服回答问题时，必须基于检索到的手册内容，并附上来源链接（可追溯）。

3. 自动化工作流

  • 场景： 每天要处理大量报表。
  • 应用：
    • 开发一个Agent。给它读取Excel的权限、发邮件的权限。
    • 让它每天自动分析数据，生成图表，并发送给老板。
