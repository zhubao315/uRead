主要内容

一、上瘾模型核心框架

上瘾模型是一个四阶段的循环，通过不断重复，将用户行为转化为无意识习惯。

习惯的定义

习惯是在几乎没有意识思考的情况下做出的行为。

习惯的商业价值

  1. 提高用户终身价值（LTV）：习惯用户更频繁使用产品
  2. 降低获客成本：用户主动返回，减少营销支出
  3. 提高定价能力：习惯用户对价格不敏感
  4. 加速增长：习惯用户更可能推荐产品

上瘾模型的四个阶段

触发 → 行动 → 多变的酬赏 → 投入 → [循环回到触发]

每个循环都强化习惯，直到用户形成自动化行为。

二、第一阶段：触发（Trigger）

触发是上瘾循环的起点，促使用户采取行动。

外部触发（External Triggers）

外部触发是环境中的提示，告诉用户下一步做什么。

四种外部触发类型：

  1. 付费型触发

    • 广告、SEM、展示广告
    • 成本高，适合新用户获取
    • 例子：Facebook广告、Google AdWords

  2. 回馈型触发

    • PR、媒体报道、应用商店推荐
    • 成本低但难以规模化
    • 例子：Product Hunt推荐、TechCrunch报道

  3. 人际型触发

    • 用户主动推荐
    • 最有效但需要产品足够好
    • 例子：朋友推荐、口碑传播

  4. 自主型触发

    • 推送通知、邮件、桌面图标
    • 需要用户授权，但效果持久
    • 例子：微信消息通知、邮件提醒

外部触发到内部触发的转变

最成功的产品最终将外部触发转化为内部触发。

内部触发（Internal Triggers）

内部触发是用户的情绪或感受。

核心洞察：人们使用产品不是为了产品本身，而是为了缓解某种不适。

常见内部触发情绪：

  • 无聊 → 刷Facebook/抖音
  • 孤独 → 打开社交应用
  • 不确定 → Google搜索
  • 焦虑 → 查看邮件/新闻

如何识别用户的内部触发：

使用”5个为什么”法：

  • 为什么用户使用你的产品？
  • 为什么他们需要这个？
  • 为什么这对他们重要？
  • （重复5次，直到找到情绪根源）

案例：Instagram

  • 表面需求：分享照片
  • 深层情绪：害怕平凡生活不被认可，渴望社交认同

三、第二阶段：行动（Action）

在触发之后，用户需要采取行动。行动阶段的关键是简化操作。

Fogg行为模型

斯坦福大学BJ Fogg教授提出：

行为 = 动机（Motivation）× 能力（Ability）× 触发（Trigger）

三个要素缺一不可：

  • 有触发但没动机或能力 → 不会行动
  • 有动机和能力但没触发 → 不会行动
  • 三者同时具备 → 行动发生

简化操作的6个维度

  1. 时间（Time）：完成行为需要多长时间？
  2. 金钱（Money）：需要多少财务支出？
  3. 体力（Physical Effort）：需要多少身体劳动？
  4. 脑力（Brain Cycles）：需要多少思考？
  5. 社会偏差（Social Deviance）：是否违背社会规范？
  6. 非常规性（Non-Routine）：是否打破现有习惯？

简化原则：找出当前阻碍用户的最大障碍，优先解决它。

案例分析：

Twitter的简化：

  • 最初功能复杂，用户不知道该做什么
  • 简化为：140字微博客 → 降低脑力和时间成本
  • 增加”What’s happening?”提示 → 降低脑力成本
  • 结果：用户更容易开始发推

Google搜索的简化：

  • 极简界面 → 降低脑力成本
  • 自动补全 → 降低体力和时间成本
  • 即时搜索 → 降低时间成本

三种核心动机

Fogg提出人类有三种核心动机：

  1. 追求快乐，逃避痛苦
  2. 追求希望，逃避恐惧
  3. 追求认同，逃避排斥

产品应该利用这些动机，而非对抗它们。

四、第三阶段：多变的酬赏（Variable Reward）

多变的酬赏是上瘾模型最强大的阶段。

为什么”多变”很重要？

神经科学研究发现：

  • 可预测的奖励 → 多巴胺分泌减少
  • 不可预测的奖励 → 持续分泌多巴胺，产生期待感

这就是为什么老虎机比ATM更让人上瘾。

三种多变酬赏类型

1. 社交酬赏（Tribe）

来自他人的认可和接纳。

案例：

  • Facebook的点赞：不知道会获得多少赞
  • Twitter的转发：不知道哪条推文会火
  • Stack Overflow的声望值：帮助他人获得认可

关键：利用人类的社交需求和认同感。

2. 猎物酬赏（Hunt）

获取资源的满足感。

案例：

  • Twitter信息流：不断滚动寻找有趣内容
  • Pinterest：寻找灵感和想法
  • 老虎机：寻找大奖

关键：满足人类收集和获取的本能。

3. 自我酬赏（Self）

掌控感、成就感、能力提升。

案例：

  • 电子邮件收件箱归零：掌控感
  • 玩游戏升级：成就感
  • 学习Duolingo：能力提升

关键：满足人类对胜任感和控制感的需求。

设计多变酬赏的原则

  1. 保持有限的可变性：完全随机会让人沮丧
  2. 让用户感觉有控制：用户相信自己的行为影响结果
  3. 避免酬赏疲劳：不要让用户每次都得到相同酬赏

警告信号：

  • 用户感觉被操控 → 反感
  • 酬赏与用户期望不符 → 流失
  • 酬赏太容易获得 → 失去价值

五、第四阶段：投入（Investment）

投入阶段要求用户付出，但不是消费，而是储存价值。

投入的两大作用

1. 增加产品价值

用户投入越多，产品对他价值越大。

案例：

  • LinkedIn：完善个人资料 → 个人资料越完整，价值越高
  • Instagram：关注朋友 → 关注列表越多，信息流越有价值
  • Pinterest：创建画板 → 画板越多，越难离开

心理学原理：

  • 宜家效应：人们高估自己创造的东西
  • 一致性原理：人们倾向于与过去行为保持一致
  • 避免损失：已经投入的东西不愿放弃

2. 加载下一个触发

投入为下一轮循环创造触发机会。

案例：

  • Twitter：关注好友 → 好友发推时触发通知
  • Facebook：上传照片 → 好友点赞时触发通知
  • Slack：加入频道 → 有新消息时触发通知

投入的类型

  1. 时间投入：学习使用产品
  2. 数据投入：输入个人信息
  3. 努力投入：创建内容
  4. 社交投入：邀请朋友
  5. 金钱投入：购买虚拟货币

设计投入的原则

  1. 时机很重要：在酬赏之后要求投入
  2. 小步快跑：不要一次要求太多
  3. 使投入有意义：确保投入真的增加价值

反例：

  • 在用户还没体验到价值前就要求注册 → 摩擦太大
  • 要求填写10页表格 → 投入过度
  • 投入后没有得到相应价值 → 用户失望

六、道德考量

艾亚尔提出”操控矩阵”（Manipulation Matrix），帮助产品设计者思考伦理问题。

操控矩阵的两个维度

  1. 产品改善用户生活吗？（是/否）
  2. 你自己会用这个产品吗？（是/否）

四种类型：

1. 促进者（Facilitator） - 会用 + 改善生活

  • 例子：健身应用、教育产品
  • 道德评价：最佳，创造真正价值

2. 商人（Peddler） - 不用 + 改善生活

  • 例子：医疗设备（医生不需要，但病人需要）
  • 道德评价：可以接受，但要谨慎

3. 娱乐者（Entertainer） - 会用 + 不改善生活

  • 例子：游戏、社交娱乐应用
  • 道德评价：灰色地带，需要审视

4. 贩毒者（Dealer） - 不用 + 不改善生活

  • 例子：赌博、成瘾性游戏
  • 道德评价：不道德，应避免

设计者的责任

艾亚尔强调：

  • 问自己：“我会给家人使用这个产品吗？”
  • 透明度：让用户理解产品如何影响他们
  • 增强自主性：给用户控制权
  • 创造真正价值：习惯应该帮助用户实现目标

核心观点

  1. 习惯是产品成功的关键：习惯用户比新用户价值高得多，培养用户习惯是最强大的增长策略

  2. 上瘾模型是可复制的框架：触发→行动→多变的酬赏→投入，四个阶段形成有效循环

  3. 内部触发比外部触发更强大：最成功的产品将使用行为与用户情绪关联

  4. 简化行动是设计核心：降低6个维度的摩擦，让行为更容易发生

  5. 多变性创造上瘾：不可预测的酬赏比固定酬赏更能激发多巴胺，产生期待感

  6. 投入储存价值：用户投入的时间、数据、努力增加产品价值，形成迁移成本

  7. 道德责任不可忽视：设计习惯养成产品时必须考虑对用户生活的真实影响

  8. 习惯测试指导迭代：通过识别习惯用户和非习惯用户的行为差异优化产品

金句摘录

“产品设计的终极目标不是让用户Think Different，而是让他们Stop Thinking。”

解析：真正成功的产品是让用户形成无意识的使用习惯，而不需要每次都做理性决策。就像刷牙一样自然。

“习惯不能被创造，只能被培养。”

解析：产品不能强迫用户形成习惯，但可以通过正确的设计引导和强化用户的自然行为，逐步培养成习惯。

“用户采取行动不是因为触发本身，而是因为触发激活了他们内心已有的需求。”

解析：触发只是点燃引线，真正驱动行为的是用户内心的情绪和需求。设计者要理解用户的深层动机。

“多变的酬赏不是随机的酬赏，而是让用户感觉自己的行为影响了结果。”

解析：纯随机让人沮丧，而给予用户控制感的可变性才能持续吸引。这就是为什么游戏中的”技能+运气”比纯运气更吸引人。

“让用户投入的最佳时机是在他们获得酬赏之后。”

解析：当用户刚刚获得满足感时，他们更愿意付出努力。就像吃完美食后更愿意给餐厅写好评。

“问问自己：我会把这个产品推荐给我在乎的人吗？如果答案是否定的，就不要做。”

解析：这是艾亚尔提出的道德检验标准。如果你不愿意让家人朋友使用，说明产品可能在操纵而非帮助用户。

应用指导

例子1：社交应用如何应用上瘾模型

场景

小李的团队开发一个职场社交应用，希望提高用户活跃度和留存率。

上瘾模型应用

阶段1：设计触发

外部触发（获取新用户）：

  • 付费型：LinkedIn广告投放
  • 回馈型：在TechCrunch发布产品故事
  • 人际型：邀请奖励机制（邀请5个同事获得高级功能）
  • 自主型：每周行业洞察邮件

内部触发（深度挖掘）：

  • 使用”5个为什么”法找到情绪根源
  • Q: 为什么用户使用职场社交？ A: 寻找工作机会
  • Q: 为什么寻找工作机会？ A: 对当前工作不满
  • Q: 为什么不满？ A: 感觉职业发展停滞
  • Q: 为什么担心发展停滞？ A: 害怕被时代抛弃
  • 核心情绪：职业焦虑

设计策略：将产品定位为”缓解职业焦虑的工具”

阶段2：简化行动

识别摩擦点：

  • 注册流程需要填写10个字段 → 脑力成本高
  • 首次使用不知道该做什么 → 脑力成本高
  • 搜索职位需要多次筛选 → 时间和体力成本高

简化措施：

  • 允许LinkedIn一键登录 → 减少时间和脑力成本
  • 新用户引导：3步完成基本档案 → 减少脑力成本
  • AI推荐职位：无需主动搜索 → 减少时间成本
  • 滑动卡片式界面：像Tinder一样简单 → 减少体力成本

阶段3：设计多变酬赏

社交酬赏：

  • 查看谁浏览了你的档案（不知道会是谁）
  • 收到面试邀请通知（不知道何时何地）
  • 行业影响力排名（可能上升或下降）

猎物酬赏：

  • 每日推送5个新职位（不知道会推什么）
  • 发现同行业的新联系人（探索未知）
  • 行业内部消息和机会（信息狩猎）

自我酬赏：

  • 完善档案进度条（掌控感）
  • 技能徽章和认证（成就感）
  • 学习新技能课程（能力提升）

阶段4：引导投入

关键时机：在用户获得酬赏后要求投入

投入类型：

  1. 数据投入：

    • 获得职位推荐后 → 要求补充工作经验
    • 查看薪资报告后 → 要求填写当前薪资

  2. 内容投入：

    • 收到点赞后 → 鼓励发布更多职场洞察
    • 获得关注后 → 鼓励回复评论

  3. 社交投入：

    • 找到感兴趣的联系人后 → 邀请更多同事加入
    • 加入行业群组后 → 邀请朋友进群

投入的价值：

  • 档案越完整 → 推荐越精准
  • 关系网越大 → 机会越多
  • 内容越多 → 影响力越大

6个月后成果：

  • 日活用户提升70%
  • 留存率从15%提升到45%
  • 用户推荐成为主要获客渠道

例子2：教育产品的习惯培养

场景

张女士开发一个在线学习平台，但用户注册后很少回来学习，完课率不到10%。

诊断问题

使用习惯测试：

  • 习惯用户定义：一周至少学习3次
  • 当前习惯用户占比：仅5%
  • 非习惯用户流失点：第二次访问

应用上瘾模型优化

优化触发

问题：依赖外部触发（邮件提醒），但打开率低

内部触发挖掘：

  • 用户为什么学习？表面：提升技能
  • 为什么提升技能？深层：职场焦虑、自我怀疑
  • 核心情绪：害怕落后、渴望成长

新策略：

  • 将课程内容与”克服焦虑”关联
  • slogan改为：“每天15分钟，甩掉职场焦虑”
  • 推送内容改为：“你的同龄人已经学完X课程了”（利用害怕落后的情绪）

简化行动

问题识别：

  • 课程视频长达60分钟 → 时间成本高
  • 需要专门找时间学习 → 非常规性高
  • 学习界面复杂 → 脑力成本高

简化方案：

  • 拆分为3-5分钟微课 → 降低时间成本
  • 支持碎片化学习（通勤、等待） → 降低非常规性
  • 极简播放界面：只有播放/暂停 → 降低脑力成本
  • 离线下载 → 降低障碍

增强多变酬赏

社交酬赏：

  • 学习排行榜（不知道自己排第几）
  • 同学点赞和评论（不知道谁会互动）
  • 结业证书社交分享（获得认可）

猎物酬赏：

  • 每日推荐新课程（发现新知识）
  • 闯关式学习路径（探索未知领域）
  • 随机知识彩蛋（意外收获）

自我酬赏：

  • 连续打卡天数（掌控感）
  • 技能树点亮（成就感）
  • 知识测试满分（能力证明）

设计投入机制

时机：完成一个微课后（刚获得满足感）

投入类型：

  1. 做笔记 → 增加记忆和归属感
  2. 提问讨论 → 社交投入
  3. 创建学习计划 → 承诺投入
  4. 推荐给朋友 → 社交投入

投入价值体现：

  • 笔记越多 → 复习资料越丰富
  • 讨论越多 → 学习社区越有价值
  • 好友越多 → 学习氛围越好

3个月后改进：

  • 完课率从10%提升到35%
  • 习惯用户从5%提升到25%
  • 用户推荐带来40%新用户

例子3：健康应用的反思——避免成为”贩毒者”

场景

王先生开发了一个计步应用，用户粘性很高，但用户反馈说”感觉被绑架了”、“不看步数就焦虑”。

道德审视

使用操控矩阵评估：

  • 我会用这个产品吗？是，自己每天都用
  • 产品改善用户生活吗？不确定，可能增加焦虑

产品定位：目前处于”娱乐者”或接近”贩毒者”区域

问题识别

  1. 过度游戏化：

    • 连续打卡天数中断后，用户极度失望
    • 排行榜让用户过度竞争
    • 每日目标未达成时产生负罪感

  2. 错误的内部触发：

    • 利用焦虑而非缓解焦虑
    • 让用户感觉被监控而非被帮助

  3. 不健康的投入：

    • 用户为了步数而无意义地走动
    • 忽视身体信号强行完成目标

重新设计（向”促进者”转型）

原则调整：

  • 从”让用户上瘾”转向”帮助用户建立健康习惯”
  • 从”竞争”转向”自我提升”
  • 从”焦虑驱动”转向”成就驱动”

具体改进：

触发优化：

  • 减少推送频率，不要过度打扰
  • 正向鼓励而非负面提醒：“今天是美好的散步天气”而非”你还没完成目标”
  • 内部触发从”焦虑”转向”活力”：“感觉困倦？5分钟散步可以提神”

行动简化：

  • 不要求每天打卡，允许休息日
  • 目标个性化，而非固定的10000步
  • 关注整体健康，而非单一指标

酬赏重设计：

  • 减少社交竞争酬赏
  • 增加自我酬赏：身体感觉改善、睡眠质量提升
  • 长期酬赏：健康数据趋势改善

投入引导：

  • 记录感受日志（而非仅看数字）
  • 设定健康目标（而非步数目标）
  • 邀请家人一起运动（健康的社交投入）

增加自主性：

  • 允许用户关闭提醒
  • 允许用户设置”休息模式”
  • 不惩罚中断，强调长期进步

6个月后转变：

  • 用户满意度从6分提升到8.5分
  • 用户焦虑感下降40%
  • 长期留存率反而提高（从30%到55%）
  • 品牌口碑改善，被健康媒体推荐

启示：

  • 高粘性≠好产品，要审视是否真正帮助用户
  • 道德设计可以与商业成功共存
  • 用户信任比短期增长更重要

例子4：电商应用的上瘾机制

场景

赵女士的电商应用获客成本高，复购率低，希望通过上瘾模型提升用户终身价值。

战略目标

从”交易型”转向”习惯型”电商

上瘾模型完整应用

阶段1：触发设计

外部触发：

  • 付费：Instagram广告（吸引新用户）
  • 自主：每日特价推送（培养每日打开习惯）

内部触发挖掘：

  • 用户为什么网购？表面：需要商品
  • 为什么需要？深层：
    • 无聊时：寻找刺激和新鲜感
    • 压力时：购物治疗，获得控制感
    • 不安全时：追求品质生活的确定感

策略：将应用与”无聊”、“压力”等情绪关联

阶段2：行动简化

降低摩擦：

  • 一键购买（免输入地址和支付信息）
  • 个性化首页（无需搜索）
  • AR试穿试用（降低决策成本）
  • 7天无理由退货（降低风险感知）

阶段3：多变酬赏

社交酬赏：

  • 好物分享获得点赞（不知道会有多少人喜欢）
  • 穿搭晒单社区（获得时尚认可）
  • “你可能认识的买家”（发现同好）

猎物酬赏：

  • 每日限时闪购（不知道今天有什么惊喜）
  • 盲盒/神秘礼物（开箱惊喜）
  • 积分抽奖（不确定能中什么）

自我酬赏：

  • 品味分数（自我认同）
  • 购物等级提升（成就感）
  • 个性化推荐越来越准（掌控感）

阶段4：投入机制

投入类型：

  1. 数据投入：

    • 完善兴趣标签 → 推荐更精准
    • 设置提醒（生日、节日）→ 未来触发

  2. 社交投入：

    • 关注喜欢的品牌/店铺 → 新品上架时触发
    • 加入兴趣社群 → 社区活跃时触发

  3. 内容投入：

    • 发布购物晒单 → 获得点赞后触发
    • 创建愿望清单 → 降价时触发

  4. 虚拟货币投入：

    • 购买会员 → 沉没成本，更难离开
    • 积分系统 → 积分越多越难放弃

投入的复利效应：

  • 关注10个品牌 → 每周收到10次新品通知
  • 加入5个社群 → 持续活跃内容
  • 积累5000积分 → 舍不得换平台

道德检查：

使用操控矩阵：

  • 我会用吗？是
  • 改善生活吗？部分是（提供便利），部分否（可能过度消费）

道德改进：

  • 增加消费提醒：“本月已消费X元，超出预算”
  • 提供”愿望清单冷静期”：加入愿望清单后7天再询问是否购买
  • 不利用负面情绪过度营销
  • 透明化推荐算法：告诉用户为什么推荐这个商品

12个月后成果：

  • 月活用户从10%提升到35%
  • 复购率从15%提升到45%
  • 用户终身价值提升3倍
  • 同时，投诉率下降（因为道德改进）
