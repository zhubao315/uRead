# 内容风格指南

uRead 的目标不是摘抄，而是产出“可读、可复用、可检索”的深度阅读资产。

## 写作原则

- 先给结论，再展开结构
- 观点必须能落到模型、方法或行动
- 尽量减少空泛赞美，增加判断依据
- 保持一篇笔记只有一个清晰主题轴

## 推荐结构

1. 一句话结论
2. 这本书在解决什么问题
3. 核心观点
4. 认知模型
5. 我的理解
6. 实践应用
7. 关联内容

## Agent 友好要求

- `summary` 不超过 80 字
- `tags` 控制在 3 到 6 个
- 关联内容尽量使用 `[[Wiki Link]]`
- 避免标题党，标题应稳定可识别
